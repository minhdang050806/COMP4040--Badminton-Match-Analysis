## Data Processing

The implementation follows the upstream BST collation contract but writes arrays incrementally with NumPy `.npy` memmaps instead of loading every clip into RAM.

For each split:

1. Visit class folders in the merged ShuttleSet class order from `get_merged_stroke_types()`.
2. Collect all `*_pos.npy` branches and verify the matching `*_joints.npy`, `*_pos.npy`, and `*_shuttle.npy` triple exists.
3. Load each clip triple.
4. Convert arrays to `float32`.
5. Pad/downsample with the upstream `make_seq_len_same(seq_len=100)` logic.
6. Generate COCO-17 bone/interpolation features with the 19 upstream COCO bone pairs.
7. Write packed arrays directly to split-level `.npy` files.
8. Write split metadata linking each row index back to source branch, class name, label id, clip id, original length, and collated video length.

Generated pose representations:

| Array | Meaning | Shape per split |
|---|---|---|
| `J_only.npy` | original 17 joints | `(N, 100, 2, 17, 2)` |
| `JnB_interp.npy` | 17 joints + 19 interpolated limb centers | `(N, 100, 2, 36, 2)` |
| `JnB_bone.npy` | 17 joints + 19 bone vectors | `(N, 100, 2, 36, 2)` |
| `Jn2B.npy` | 17 joints + 19 limb centers + 19 bone vectors | `(N, 100, 2, 55, 2)` |
| `pos.npy` | normalized player positions | `(N, 100, 2, 2)` |
| `shuttle.npy` | normalized shuttle coordinates | `(N, 100, 2)` |
| `videos_len.npy` | valid unpadded sequence length | `(N,)` |
| `labels.npy` | merged class id | `(N,)` |

## Assumptions and Unclear Items

- The selected source variant is `merged_seq100_between_2_hits_with_max_limits` because Phase 04's recommended checkpoint is the merged 25-class, 2D, `JnB_bone`, seq100 BST checkpoint.
- Phase 03 does not collate the 35-class original labels or 3D variants.
- Phase 03 does not use the full 36,482-row Phase 02 annotation table as the source of truth for features. It uses the 33,481 feature branches confirmed in Phase 01.
- Unclear from current codebase: why 3,001 Phase 02 annotation rows do not have corresponding feature clips.
- Unclear from current codebase: whether Phase 04 should evaluate only the test split or also export train/val predictions for diagnostics. This phase prepared all three splits so either choice is possible.
