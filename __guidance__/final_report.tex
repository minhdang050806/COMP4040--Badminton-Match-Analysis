\begin{abstract}
Badminton broadcasts contain tactical information that is useful for coaching
and match review, but this information is not directly available as data. A
system must first find rally footage, track the shuttle, locate both players,
identify stroke moments, classify stroke types, and then organize the result
as an ordered rally sequence. We present \method, an end-to-end badminton video
analytics and data mining pipeline built on the local ShuttleSet data. The
pipeline combines SA-CNN rally filtering, TrackNetV3 shuttle tracking,
YOLO26x-pose player extraction, court normalization, a fine-tuned
Badminton-Stroke Transformer, and five tactical mining analyses: stroke
distributions, Markov transitions, frequent contiguous motifs,
movement-strategy profiling, and KMeans clustering. On unseen ShuttleSet
videos 41--44, the integrated system produces 3,192 stroke predictions; after
removing 14 simultaneous-event alias rows, the clean benchmark contains 3,178
predictions with 69.04\% side-aware top-1 accuracy, 87.89\% top-3 accuracy,
93.05\% player-side accuracy, and 66.64\% supported macro-F1. More
importantly for data mining, the predicted \emph{none} rate falls from 56.43\%
in an earlier integrated baseline to 6.70\%, transition Frobenius error falls
by 66.3\%, and mean row KL divergence falls by 95.6\%. The recovered sequences
preserve a clear attack--defense grammar around net shots, lifts, smashes, and
defensive replies. Movement analysis gives cautious descriptive evidence about
movement and attack choices, while clustering remains too weak to support a
stable player-style taxonomy.
\end{abstract}

\section{Introduction}

Badminton is a fast racket sport where a point can change direction within a
few strokes. Coaches and analysts often want to answer questions such as:
which strokes dominate a match, what replies usually follow a smash, whether a
player creates attacks after moving forward, and whether two players show
different tactical profiles. These questions are naturally data mining
questions, but broadcast video does not provide clean tables of strokes and
positions. The data must first be recovered from pixels.

This makes raw-video badminton analysis difficult. The shuttle is small,
motion-blurred, and often occluded. Broadcast videos include replays,
close-ups, scoreboards, and non-rally segments. The far-side player can be
small and partially missing. Stroke labels are also fine-grained: a system
must distinguish, for example, a net shot from a return net, a drop from a
push, and a smash from related attacking strokes. A tactical analysis based on
these predictions must therefore be honest about error propagation. A visually
plausible model is not enough; the final event stream must remain coherent
enough for transition and pattern mining.

The objective of this project is to build and evaluate a practical end-to-end
pipeline that converts badminton broadcast video into mineable tactical data.
The central research question is:

\begin{quote}
\emph{Can an integrated raw-video pipeline recover badminton stroke sequences
that are accurate and continuous enough to support meaningful tactical mining
on unseen matches?}
\end{quote}

Our answer is cautiously positive. The current pipeline is not a fully
automatic arbitrary-video system, because the unseen-video benchmark still
uses ShuttleSet hit-frame labels and homography metadata for construction and
evaluation. However, it does process unseen raw frames through shuttle
tracking, pose extraction, feature construction, stroke classification, and
tactical mining. It produces a useful predicted tactical baseline and clearly
identifies the main bottleneck: far-side player extraction.

The main contributions are:
\begin{itemize}[leftmargin=*,nosep]
  \item an integrated raw-video badminton analytics pipeline that connects
  rally filtering, shuttle tracking, pose extraction, stroke classification,
  and tactical mining;
  \item a clean unseen-video evaluation on ShuttleSet videos 41--44 with
  3,178 evaluated stroke predictions and 69.04\% side-aware accuracy;
  \item a result-oriented analysis showing that sequence quality improved more
  than top-1 accuracy alone suggests, because the \emph{none} rate and
  transition errors were greatly reduced;
  \item five data mining analyses that explain stroke composition, local
  transitions, repeated motifs, movement profiles, and exploratory clusters;
  \item a practical diagnosis that far-side pose quality controls downstream
  reliability more strongly than the choice of mining algorithm.
\end{itemize}

\section{Related Work}

\paragraph{Badminton datasets.}
ShuttleSet provides broadcast videos, set-level annotations, stroke labels,
homography information, and pre-extracted features for badminton analysis
\cite{shuttleset}. It is useful for this project because it contains both raw
videos and structured labels, so we can evaluate how much tactical structure
survives after the pipeline predicts from video. BadmintonDB focuses on
player-specific badminton match analysis and prediction \cite{badmintondb}.
Our work uses ShuttleSet as the main benchmark because its annotations allow
stroke-level, rally-level, and sequence-level evaluation.

\paragraph{Rally filtering, shuttle tracking, and hit detection.}
Rally filtering separates playable badminton footage from non-rally broadcast
segments. Shuttle tracking is commonly formulated as frame-level heatmap or
coordinate prediction. TrackNetV2 introduced an efficient shuttle tracking
network for badminton-style video \cite{tracknetv2}, and the local project
uses a TrackNetV3 implementation for shuttle coordinate extraction and
denoising. Hitting-event detection has also been studied directly from video
clips \cite{chen2023hitting}. In this project, exact ShuttleSet hit labels are
used for the current unseen-video benchmark, while trajectory-based hit
detection remains the required step for fully unlabeled deployment.

\paragraph{Pose estimation and stroke classification.}
Player pose provides body keypoints and player position information. MMPose is
a general pose-estimation toolbox \cite{mmpose}; the active pipeline uses a
YOLO26x-pose checkpoint because it is efficient for the current video feature
extraction route. For stroke recognition, the Badminton-Stroke Transformer
(BST) models racket-sport strokes using player skeletons, positions, and
shuttle features \cite{chang2025bst}. We fine-tune the BST \texttt{BST\_CG\_AP}
variant and use it as the final stroke classifier.

\paragraph{Sports tactic mining.}
Sports tactic mining often begins from structured event sequences. Stroke
distributions summarize what happened, transition matrices explain what tends
to happen next, and frequent motifs reveal repeated tactical phrases.
Movement profiles connect location and displacement with stroke choices.
KMeans clustering \cite{kmeans} and silhouette analysis \cite{silhouette} can
summarize profiles, but clusters should only be interpreted when they are
stable and well separated. The key difference in this project is that the
mining methods are applied to predicted event streams from video, not only to
manual annotations.

\section{Dataset Description and Preprocessing}

\subsection{Dataset Source and Size}

The local project uses ShuttleSet, a badminton dataset with broadcast videos
and stroke-level annotations. The current local snapshot contains 44 raw
broadcast videos, 104 set annotation files, 36,482 annotated strokes, and
3,683 annotated rallies. It also contains six pre-extracted feature variants,
each with 33,481 prepared clips. The active BST feature variant uses
side-aware merged classes and a fixed sequence length of 100 frames.

Videos 1--39 are used for development, training, and parameter selection.
Videos 41--44 are used as the project-level unseen-video evaluation set. Video
40 appears in intermediate shuttle and stroke-window outputs but is not part
of the final integrated evaluation. The inherited reference split historically
places video 41 in validation and videos 42--44 in test. For transparency, the
report gives the headline result on videos 41--44 and also reports the
stricter 42--44 subset where relevant.

\subsection{Annotation Types and Label Preparation}

The project prepares several forms of labels and metadata:
\begin{itemize}[leftmargin=*,nosep]
  \item stroke labels, including the hitter side and stroke type;
  \item rally identifiers and event order within each rally;
  \item exact hit-frame labels used to construct stroke windows in the current
  benchmark;
  \item homography metadata used to map image coordinates into a court-aware
  coordinate system;
  \item player identity information from ShuttleSet, used only for evaluation
  and movement profiling.
\end{itemize}

The final stroke classifier predicts 25 side-aware classes:
\begin{quote}
\texttt{none} + 12 Top-side stroke classes + 12 Bottom-side stroke classes.
\end{quote}
Top and Bottom are camera-relative sides, not permanent player identities.
This distinction matters for interpretation. A Top-side smash means that the
stroke was made from the far/camera-top side in that view; it does not mean
that a particular athlete is always a ``Top'' player.

\subsection{Preprocessing Pipeline}

The preprocessing converts long broadcast videos into short, aligned examples
that can be classified and mined. First, rally filtering keeps likely play
segments and removes most non-rally footage. Second, shuttle tracking produces
frame-level shuttle coordinates and applies denoising to reduce implausible
jumps and fill short gaps. Third, stroke windows are constructed around
hit-frame labels for the current unseen-video benchmark. Fourth, player pose
and position features are extracted in each window. Finally, pose, position,
and shuttle arrays are normalized into the fixed BST input format.

For each stroke window, the final feature contract is:
\begin{itemize}[leftmargin=*,nosep]
  \item player keypoints with two players and 17 body joints;
  \item two player position tracks;
  \item shuttle coordinates;
  \item validity masks that record which player features were observed or
  recovered.
\end{itemize}

The important practical point is that preprocessing does not merely prepare
data for a classifier. It also creates the quality metadata used later to
explain why some videos and strokes are more reliable than others.

\section{Modeling}

This section describes each modeling phase by its role in the full system, its
input and output, whether it is fine-tuned or inference-only, and what the
current results show.

\subsection{SA-CNN Rally Scenes Filtering}

\paragraph{Role in the pipeline.}
Rally filtering is the first stage that makes the raw broadcast video usable.
Broadcast badminton contains score screens, close-ups, replays, player
preparation, and audience shots. If these frames are sent directly to shuttle
tracking and pose estimation, downstream models waste computation and can
produce false events. SA-CNN is used as a binary visual filter that separates
court/rally scenes from non-play footage.

\paragraph{Input, output, and training.}
The model receives sampled video frames and predicts whether they belong to a
court-view rally scene. It is fine-tuned as a two-class classifier using the
prepared SA-CNN dataset. The protocol model uses 20,400 training images and
3,000 validation images. The best validation F1 is 0.903 after 20 epochs.

\paragraph{Result and interpretation.}
On the latest coverage smoke test, the filter keeps 97.32\% of annotated hits
inside accepted intervals and overlaps 100\% of annotated rallies. This is a
good sign: the filter usually keeps the play that matters. However, visual
review still shows that some accepted intervals extend into post-rally
close-ups. Therefore, the rally filter is strong enough for the current
benchmark, but precise interval boundaries remain a future deployment issue.

\subsection{TrackNetV3 Shuttle Tracking}

\paragraph{Role in the pipeline.}
Shuttle tracking estimates where the shuttle is in each rally frame. This
information is needed because stroke classification depends not only on player
pose, but also on shuttle movement and timing. The shuttle is one of the
hardest objects in badminton video: it is small, fast, and frequently blurred.

\paragraph{Input, output, and training status.}
TrackNetV3 receives accepted rally frames and predicts shuttle coordinates.
The project uses an existing TrackNetV3 checkpoint for inference and applies a
post-processing step that removes implausible jumps and interpolates short
missing segments. In the current project state, this stage is inference-based;
full TrackNet fine-tuning is not claimed because complete frame-level
shuttle-supervision for the development corpus has not been proven.

\paragraph{Result and interpretation.}
For videos 40--44, the raw visible-frame coverage is 86.63\%. After denoising,
coverage rises to 90.99\%. The denoising step removes 19,384 implausible jump
points, interpolates 27,953 short-gap points, and produces no visible
out-of-range coordinates. These numbers show that the shuttle tracker is
operational and useful, but they should not be confused with perfect tracking.
A frame can contain a visible shuttle prediction while still having a boundary
or localization error.

\subsection{Badminton Stroke-Type Transformer}

\paragraph{Role in the pipeline.}
The Badminton-Stroke Transformer is the final classifier that turns extracted
features into stroke labels. It receives the local motion context around a hit
and predicts a side-aware stroke class. This is the component that makes the
event table useful for tactical mining.

\paragraph{Input, output, and fine-tuning.}
For each stroke, the classifier receives a length-100 sequence containing
player body keypoints, player positions, and shuttle coordinates. The selected
model is the \texttt{BST\_CG\_AP} variant initialized from the available
ShuttleSet checkpoint and fine-tuned on the collated feature set. The training
split contains 25,741 clips and the validation split contains 4,241 clips. The
best validation macro-F1 is 0.805 at epoch 6, with validation accuracy around
82.74\%.

\paragraph{Result and interpretation.}
The classifier performs well on reference-quality features. The harder
problem is transferring the classifier to features extracted again from raw
unseen video. In the final integrated evaluation, the clean videos 41--44
result is 69.04\% side-aware top-1 accuracy and 87.89\% top-3 accuracy. The
system is much better at identifying the hitter side than the exact fine
stroke type: player-side accuracy is 93.05\%, while full side-aware accuracy
is 69.04\%. This means the pipeline usually knows which side made the stroke,
but still confuses related actions such as net shot, return net, drop, push,
rush, and some service continuations.

\subsection{Pose Estimation}

\paragraph{Role in the pipeline.}
Pose estimation extracts player body keypoints and approximate player
positions. These features are important because the stroke classifier needs to
know the player's body motion and court location, not only the shuttle
position. Pose is also the main source for movement-strategy analysis.

\paragraph{Input, output, and training status.}
The active system uses YOLO26x-pose on the stroke-window frames. It exports two
ordered player streams, corresponding to Top and Bottom camera-relative sides,
plus validity masks that record missing or recovered values. This stage is
inference-only in the current project; the pose model is not fine-tuned here.

\paragraph{Result and interpretation.}
The pose stage exports 3,192 feature bundles over 183,007 processed frames.
It has zero player-side inconsistent frames, which means Top/Bottom ordering
is stable. The both-player pose-missing rate is only 0.85\%, but this number
can be misleading because the missingness is highly asymmetric. The Top
far-side player has only 54.29\% pose validity, while the Bottom near-side
player has 98.96\% pose validity. This asymmetry is the most important
reliability issue in the system.

\subsection{Integration: Pose + Stroke Classification}

\paragraph{Role in the pipeline.}
Integration is the step that turns separate model outputs into a structured
data table. For every stroke window, it aligns the hit frame, rally identity,
shuttle track, player pose, player position, feature quality, predicted class,
and prediction confidence. This table is the shared input to all tactical
mining methods.

\paragraph{Input and output.}
The integration stage receives Phase 09 feature bundles and runs the
fine-tuned BST checkpoint. It outputs one ordered row per predicted stroke:
video id, rally id, event rank, event frame, predicted player side, predicted
stroke type, confidence, top alternatives, and feature-quality fields.

\paragraph{Result and interpretation.}
The integrated system produces 3,192 predictions on videos 41--44. Fourteen
rows are excluded from the clean result because simultaneous annotations share
a frame and therefore share an overwritten feature source. The clean 3,178-row
result is the headline benchmark. Per-video accuracy varies strongly:
73.68\% for video 41, 63.97\% for video 42, 53.27\% for video 43, and 76.57\%
for video 44. Video 43 is weakest, matching the observation that far-side
feature quality is poor in that domain.

\section{Data Mining Tactics}

The modeling stages produce the structured stroke stream; the data mining
stages explain what that stream says about badminton tactics. Ground-truth
findings describe the annotated ShuttleSet matches. Predicted findings
describe what the deployable video pipeline can currently recover.

\subsection{Stroke Distribution Analysis}

\paragraph{Input and features.}
Stroke distribution analysis uses the integrated predicted stroke table from
Phase 10. Its main features are predicted stroke type, predicted hitter side,
video id, rally id, and prediction confidence.

\paragraph{Processing and metrics.}
The method counts how often each stroke appears overall, by video, and by
Top/Bottom side. It also reports the \emph{none} rate because too many
\emph{none} predictions would break later sequence mining. The clean predicted
stream contains 3,178 strokes over 343 rallies, with a mean predicted rally
length of 9.27 strokes. This is close to the ground-truth mean rally length of
9.91 over all ShuttleSet annotations.

\paragraph{Results and insight.}
The predicted \emph{none} rate is 6.70\%, compared with 56.43\% in the older
integrated baseline. This is one of the most important project results because
it means the event stream is continuous enough for transition and motif
analysis. The most frequent predicted Bottom-side strokes are return net,
drop, clear, lob, and push. For Top, smash, lob, clear, net shot, and drop are
most common. These distributions summarize match composition, but they should
not be interpreted as fixed player styles because Top and Bottom are camera
sides and Top pose quality is much weaker.

\subsection{Markov Transition Modeling}

\paragraph{Input and features.}
Markov transition modeling uses the ordered stroke sequence within each rally.
Each adjacent pair of strokes becomes one transition, such as
\emph{lob $\rightarrow$ smash} or \emph{smash $\rightarrow$ return net}.

\paragraph{Processing and validation.}
The method builds a row-normalized transition matrix: each row asks what
stroke tends to come next after the current stroke. We compare three predicted
estimators against the exact-label transition matrix. The hard estimator uses
the top-1 predicted label. The soft estimator uses the full probability
vector. The high-confidence estimator keeps only adjacent strokes whose
confidence is at least 0.8. The metrics are Frobenius error and mean row KL
divergence; lower values mean the predicted transition structure is closer to
the exact-label structure.

\paragraph{Results and insight.}
The current hard transition matrix reduces Frobenius error by 66.3\% and mean
row KL by 95.6\% relative to the earlier baseline. This proves that the new
event stream is not only more accurate per stroke; it also preserves
sequence-level tactical structure. Hard predictions outperform soft
probabilities because the probability mass over secondary classes blurs strong
relations. High-confidence transitions have slightly lower Frobenius error,
but they keep only 35.59\% of strokes and fragment complete rallies.

\subsection{Frequent Contiguous Motif Mining}

\paragraph{Input and features.}
Motif mining uses the same ordered rally sequences as transition modeling, but
looks for repeated contiguous stroke phrases longer than one transition. For
example, a four-stroke motif can describe pressure, lift, attack, and defense
as one tactical unit.

\paragraph{Processing and metrics.}
The method enumerates repeated contiguous stroke sequences and reports rally
support: the fraction of rallies containing the motif at least once. This
support measure is easier to interpret than raw counts because long rallies
could otherwise dominate the result.

\paragraph{Results and insight.}
The ground-truth stream shows a recurring attack--defense grammar:
\[
\text{net shot}\rightarrow\text{lob}\rightarrow
\text{smash}\rightarrow\text{defensive net}.
\]
The full four-stroke motif appears in 11.16\% of annotated rallies. Its
sub-patterns are even stronger: \emph{net shot $\rightarrow$ lob} appears in
48.76\% of rallies, and \emph{smash $\rightarrow$ defensive net} appears in
30.30\%.

The predicted stream recovers the same idea in side-aware form. The motif
\emph{Top net shot $\rightarrow$ Bottom lob $\rightarrow$ Top smash
$\rightarrow$ Bottom return net} appears in 4.66\% of predicted unseen-video
rallies. Shorter predicted components are stronger: \emph{Top smash
$\rightarrow$ Bottom return net} appears in 28.86\%, \emph{Bottom lob
$\rightarrow$ Top smash} in 18.08\%, and \emph{Top net shot $\rightarrow$
Bottom lob $\rightarrow$ Top smash} in 7.58\%. This means the pipeline
recovers the tactical shape of the rally, although longer motifs are still
fragmented by classification errors.

\paragraph{Opening-play insight.}
Using ground-truth outcome labels only, openings show that the third shot
after service matters. \emph{Short service $\rightarrow$ push $\rightarrow$
clear} has a 37.50\% server win rate, while \emph{short service
$\rightarrow$ lob $\rightarrow$ smash} has 53.52\% and \emph{short service
$\rightarrow$ lob $\rightarrow$ tap smash} has 60.00\%. This is a tactical
finding from annotations, not yet a fully predicted result, because the
current pipeline does not predict point winner.

\subsection{Movement-Strategy Profiling}

\paragraph{Input and features.}
Movement-strategy profiling uses pose and position outputs from Phase 09
together with stroke events. Homography metadata maps valid player positions
into a normalized, player-centric half-court. This removes the most obvious
camera-side effect before comparing movement behavior.

\paragraph{Processing and validation.}
For each player-set profile with enough observations, the method summarizes
where the player hits, how much the player moves between stroke events, and
which strokes are associated with those movements. The current movement
analysis uses 2,300 valid events and 1,710 valid displacement pairs to build
16 player-set profiles; four sparse profiles are excluded.

\paragraph{Results and insight.}
The event positions concentrate around central court regions. The mean lateral
coordinate is 0.496 and the mean forward coordinate is 0.450. About 58.43\%
of events lie in the central lateral corridor, 71.22\% lie in the middle-depth
band, and 46.65\% lie in both central regions. This is consistent with
central-base positioning at stroke time.

\begin{figure}[t]
  \centering
  \includegraphics[width=\linewidth]{general_player_centric_occupancy.png}
  \caption{\textbf{Player-centric event-position occupancy.} Valid stroke
  events concentrate near central regions after camera-side normalization.
  These are event positions, not continuous footwork tracks.}
  \label{fig:occupancy}
\end{figure}

The movement correlations are cautious rather than definitive. Netward
displacement is positively associated with predicted smash rate
($r=0.304$), while right-baselineward displacement is negatively associated
with predicted smash rate ($r=-0.335$). This suggests that moving forward is
more often connected with attacking opportunities, while retreating toward
the right baseline is less connected with immediate smashing. These are
descriptive associations only; they do not prove causality.

\subsection{KMeans Clustering and Stability Analysis}

\paragraph{Input and features.}
KMeans uses profile-level features rather than individual strokes. Three views
are evaluated: ground-truth player-match profiles, predicted video-side
profiles, and extended player-set movement profiles.

\paragraph{Processing and validation.}
The profiles are standardized and clustered with KMeans. Candidate values of
$k$ are compared using silhouette score, minimum cluster size, and bootstrap
adjusted Rand index (ARI). These validation metrics are important because a
cluster plot can look interesting even when the groups are unstable.

\paragraph{Results and insight.}
The clustering result is intentionally conservative. Ground-truth
player-match profiles select $k=2$ over 88 units, but the silhouette is only
0.152 and bootstrap ARI is 0.440. Predicted video-side profiles select a
balanced $k=2$ solution over eight units with silhouette 0.200 and ARI 0.505,
but the groups mostly separate videos 42--43 from videos 41 and 44, which is
more likely to reflect video domain or perception quality than true player
style. Extended movement profiles select $k=3$ with silhouette 0.151 and ARI
0.365.

The conclusion is that clustering is useful for summarizing and diagnosing the
profiles, but the current data does not support a strong vocabulary of
universal player-style groups.

\section{Discussion}

The strongest result is not simply that the classifier reaches 69.04\%
side-aware accuracy. The more important result is that the predicted sequence
becomes coherent enough for tactical mining. The older integrated baseline
predicted \emph{none} for 56.43\% of strokes, which destroyed rally
continuity. The current pipeline reduces this to 6.70\%, allowing meaningful
Markov transitions and frequent motifs to reappear.

The recovered tactical story is reasonable for badminton singles. Net pressure
often leads to a lift or lob. A lift creates an attacking opportunity. A smash
often produces a defensive net reply rather than immediately ending the
rally. The repeated pattern
\[
\text{net shot}\rightarrow\text{lob}\rightarrow
\text{smash}\rightarrow\text{defensive reply}
\]
is visible in the ground-truth stream and partially recovered by predictions.
This is exactly the type of result that a tactical mining pipeline should
produce: not just a label table, but a readable explanation of rally structure.

The results also show where the system should improve next. Player-side
accuracy is high at 93.05\%, so the system usually knows which side hit the
shot. The main errors are fine stroke distinctions and feature quality,
especially for the far-side player. On the stricter 42--44 subset, rows with
Top pose validity at least 75\% reach 76.71\% accuracy, while rows below 50\%
Top pose validity reach only 56.05\%. This 20.66 percentage-point gap is
larger than any gain suggested by changing the downstream mining method.

The mining results should therefore be interpreted in layers. Stroke
distributions and transition matrices are useful predicted outputs. Frequent
motifs are meaningful when they are short or strongly supported, but longer
motifs are sensitive to one wrong label. Movement profiles are helpful for
descriptive analysis, especially central positioning and movement--attack
associations. KMeans is the weakest as a tactical conclusion: it helps explore
profiles but does not justify naming stable player archetypes.

\section{Limitation and Future Work}

The first limitation is the benchmark boundary. The current unseen-video
experiment uses raw frames for shuttle tracking, pose extraction, feature
generation, and stroke classification, but it still uses ShuttleSet exact hit
labels to construct stroke windows and ShuttleSet homographies for court
normalization. Therefore, the result should be described as an unseen-video
raw-feature and classification benchmark with supplied temporal and court
metadata, not as a fully automatic system for arbitrary unlabeled broadcasts.

The second limitation is perception quality. Far-side Top-player pose
validity is only 54.29\%, compared with 98.96\% for the near-side Bottom
player. This affects stroke classification and any movement result that uses
player position. Rally boundaries also require more work, because accepted
intervals can still include post-rally close-ups.

The third limitation is output scope. The current system does not predict
point winner, win/loss reason, exact shuttle landing location, persistent
player identity in arbitrary unseen video, or continuous footwork. Opening
win-rate analysis is therefore ground-truth-only. Approximate shuttle endpoint
visualizations should not be interpreted as true landing positions.

Future work should focus on:
\begin{itemize}[leftmargin=*,nosep]
  \item improving far-side player detection and tracking;
  \item evaluating trajectory-based hit detection for fully unlabeled videos;
  \item adding user-assisted or automatic court calibration for new broadcasts;
  \item tightening rally boundary detection and visual QA;
  \item improving fine-grained stroke discrimination for net, drop, push,
  rush, and service-related classes;
  \item adding point-winner and outcome prediction before making causal
  tactical claims;
  \item testing the pipeline on more unseen matches beyond the supplied
  ShuttleSet metadata.
\end{itemize}

\section{Conclusion}

This project builds \method, an integrated badminton video analytics and data
mining system that converts broadcast video into structured stroke sequences
and then mines those sequences for tactical patterns. The system combines
rally filtering, shuttle tracking, player pose extraction, court-normalized
feature construction, a fine-tuned stroke transformer, and five interpretable
mining methods.

On unseen ShuttleSet videos 41--44, the final clean evaluation reaches
69.04\% side-aware top-1 accuracy, 87.89\% top-3 accuracy, and 66.64\%
supported macro-F1. More importantly, it reduces the predicted \emph{none}
rate to 6.70\% and greatly improves transition-matrix quality relative to the
earlier integrated baseline. This allows the predicted stream to recover a
reasonable badminton attack--defense grammar involving net pressure, lifts,
smashes, and defensive replies.

The project also clarifies what should and should not be claimed. Stroke
distributions, hard Markov transitions, common motifs, and event-position
profiles are useful predicted analyses. Movement correlations are descriptive,
not causal. KMeans clusters are exploratory and not stable player archetypes.
The strongest next improvement is better far-side player perception, followed
by fully unlabeled hit detection and new-video court calibration.
